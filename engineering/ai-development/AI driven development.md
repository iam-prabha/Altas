# The 7 Phase of AI-driven development

## Idea
- Have some kind of for an app, features or refactor.

## Research (optional)
- If execution would involve difficult 'explore' phases, create a **research.md** asset to cache them

## prototype (optional)
- Hash out ideas in code to get early feedback. provide assests that can later be used in the implementation.

## product requirement document (PRD)
- create a markdown file to describe the destination with user stories and implementation notes.

## Kanban
- turn the PRD into individual tickets with blocking relationships.

## Execution
- In a loop, run a coding agent to execute all of the tickets on the kanban board.

## QA
- create a QA plan for human to QA the completed work.
