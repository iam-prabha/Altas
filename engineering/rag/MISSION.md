# Mission: Learn RAG

## Why am I learning this?

**Two equally important drivers:**

1. **Building a product** — I want to build something real with RAG. I don't know exactly what yet, but I want the skills to prototype, evaluate, and ship a RAG-powered application.

2. **Getting a job** — RAG is a high-signal topic in AI/ML engineering interviews. I need to understand it deeply enough to answer architectural questions, discuss tradeoffs, and evaluate systems.

## Constraints

- **Beginner** — I'm starting from zero on RAG specifically (though I know Python)
- **Python** — my primary language
- **45–60 min sessions** — lessons should fit in one focused hour
- **Flexible tooling** — I'm open to Ollama (local) or OpenAI API for the LLM; don't lock into one

## What success looks like

- I can build a RAG pipeline from scratch (no framework magic)
- I can explain chunking, embeddings, retrieval, and generation tradeoffs in an interview
- I can evaluate a RAG system's quality
- I can evolve from "naive RAG" to more advanced patterns
- I can decide what product to build once I know what's possible
