RAG from Scratch

Build your first working Retrieval-Augmented Generation pipeline — with no frameworks, just Python.

What you’ll build

By the end of this lesson you’ll have a rag.py that:

  1. Loads a document and chunks it
  2. Turns each chunk into an embedding (a vector)
  3. Searches those vectors to find relevant chunks for your question
  4. Generates an answer using an LLM with the retrieved context

What is RAG?

Retrieval-Augmented Generation is a pattern that gives an LLM access to external knowledge at inference time. Instead of answering from its training data alone, the model first retrieves relevant documents, then generates an answer grounded in those documents.

Think of it as giving the LLM an open-book exam: it can look up facts before answering. Without RAG, it’s answering from memory alone — which is why it hallucinates or gives outdated information.

The three steps every RAG system follows:

Step Name What happens
1 Index Split documents into chunks, embed each chunk into a vector, store in a searchable index
2 Retrieve Embed the user’s query, find the most similar chunks via vector search
3 Generate Feed the query + retrieved chunks to an LLM, get a grounded answer

This is called “Naive RAG” — just the three steps in sequence. More advanced variants add pre/post-processing, but this is the core every system builds on.

Interview prep: You’ll often be asked to “walk through how a RAG system works.” The Index → Retrieve → Generate framework is your mental model. Know it cold.

Setup

Install dependencies:

pip install sentence-transformers faiss-cpu numpy openai requests

If you want to use a local LLM, install Ollama and pull a model:

ollama pull llama3.2

Create a file called sample.txt with any text you want to query against. Here’s a short one to start:

Retrieval-Augmented Generation (RAG) is a technique that combines
information retrieval with text generation. It works by first retrieving
relevant documents from a knowledge base, then feeding those documents
as context to a language model to generate a grounded response.

One of the key advantages of RAG is that it reduces hallucinations by
grounding the model's output in retrieved evidence. It also enables
the system to access up-to-date information without retraining.

RAG systems typically use embeddings to represent documents and
queries as vectors, then perform similarity search to find the most
relevant documents for a given query. The retrieved documents are
then concatenated with the query and passed to the LLM for generation.

Step 1: Chunking

Step 1

Documents are too long to fit in an LLM’s context window. We split them into smaller pieces (chunks) that can be retrieved and fed to the model.

A simple chunking strategy: split by character count with overlap so no meaning is lost at boundaries.

def chunk_text(text, chunk_size=500, overlap=50):
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks

with open("sample.txt") as f:
    text = f.read()

chunks = chunk_text(text)
print(f"Split into {len(chunks)} chunks")
for i, c in enumerate(chunks):
    print(f"--- Chunk {i} ({len(c)} chars) ---")
    print(c[:100] + "...")

Step 2: Embeddings

Step 2

Computers don’t understand text — they understand numbers. An embedding is a list of numbers (a vector) that captures the meaning of a piece of text. Similar texts have similar vectors.

from sentence_transformers import SentenceTransformer

# Load a small, fast embedding model
model = SentenceTransformer("all-MiniLM-L6-v2")

# Turn chunks into vectors
chunk_embeddings = model.encode(chunks)
print(f"Embedding shape: {chunk_embeddings.shape}")
# (num_chunks, 384) — each chunk becomes a 384-dimensional vector

all-MiniLM-L6-v2 is a popular embedding model. It’s small (80 MB), fast, and gives decent quality. You can swap it for stronger models (e.g., BAAI/bge-large-en-v1.5) later.

Step 3: Vector Search with FAISS

Step 3

Now we need to find the chunks most relevant to a user’s question. We embed the question, then find the chunks whose vectors are closest to it. FAISS (Facebook AI Similarity Search) does this efficiently.

import faiss
import numpy as np

# Build a FAISS index
dimension = chunk_embeddings.shape[1]  # 384
index = faiss.IndexFlatL2(dimension)
index.add(chunk_embeddings)

def retrieve(query, k=2):
    # Embed the query
    query_vec = model.encode([query])
    # Search
    distances, indices = index.search(query_vec, k)
    results = [chunks[i] for i in indices[0]]
    return results

# Try it!
query = "How does RAG reduce hallucinations?"
results = retrieve(query)
print("Retrieved chunks:")
for r in results:
    print(f"  • {r[:80]}...")

L2 (Euclidean distance) is the simplest similarity metric. In practice, cosine similarity is more common for text embeddings — we’ll cover that in a later lesson.

Step 4: Generation

Step 4

Finally, we feed the retrieved context + the original question to an LLM. The LLM uses the context to generate a grounded answer.

Option A: Ollama (local, free)

import requests
import json

def generate_ollama(query, context, model="llama3.2"):
    prompt = f"""Answer the question using ONLY the provided context.

Context:
{chr(10).join(context)}

Question: {query}

Answer:"""
    response = requests.post("http://localhost:11434/api/generate",
        json={"model": model, "prompt": prompt, "stream": False})
    return response.json()["response"]

# Usage
context = retrieve(query)
answer = generate_ollama(query, context)
print("Answer:", answer)

Option B: OpenAI API

from openai import OpenAI

client = OpenAI()  # requires OPENAI_API_KEY env var

def generate_openai(query, context, model="gpt-4o-mini"):
    prompt = f"Answer the question using ONLY the provided context.\n\nContext:\n{chr(10).join(context)}\n\nQuestion: {query}\n\nAnswer:"
    response = client.chat.completions.create(
        model=model,
        messages=[{"role": "user", "content": prompt}]
    )
    return response.choices[0].message.content

# Usage
answer = generate_openai(query, context)
print("Answer:", answer)

The prompt instructs the LLM to “answer using ONLY the provided context.” This is critical — without it, the model may fall back on its training data and hallucinate.

Putting It All Together

Here’s the full rag.py:

import requests
import json
import os
import numpy as np
from sentence_transformers import SentenceTransformer
import faiss

# --- Step 1: Load & Chunk ---
def load_and_chunk(path, chunk_size=500, overlap=50):
    with open(path) as f:
        text = f.read()
    chunks = []
    start = 0
    while start < len(text):
        end = min(start + chunk_size, len(text))
        chunks.append(text[start:end])
        start += chunk_size - overlap
    return chunks

# --- Step 2: Embed ---
print("Loading embedding model...")
model = SentenceTransformer("all-MiniLM-L6-v2")
chunks = load_and_chunk("sample.txt")
chunk_embeddings = model.encode(chunks)
print(f"Indexing {len(chunks)} chunks...")

# --- Step 3: Index & Search ---
dimension = chunk_embeddings.shape[1]
index = faiss.IndexFlatL2(dimension)
index.add(chunk_embeddings)

def retrieve(query, k=2):
    query_vec = model.encode([query])
    distances, indices = index.search(query_vec, k)
    return [chunks[i] for i in indices[0]]

# --- Step 4: Generate ---
def generate(query, context, backend="ollama"):
    if backend == "ollama":
        prompt = f"Answer using ONLY the context.\n\nContext:\n{chr(10).join(context)}\n\nQuestion: {query}\n\nAnswer:"
        resp = requests.post("http://localhost:11434/api/generate",
            json={"model": "llama3.2", "prompt": prompt, "stream": False})
        return resp.json()["response"]
    else:
        from openai import OpenAI
        client = OpenAI()
        prompt = f"Answer using ONLY the context.\n\nContext:\n{chr(10).join(context)}\n\nQuestion: {query}\n\nAnswer:"
        resp = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}]
        )
        return resp.choices[0].message.content

# --- Run ---
if __name__ == "__main__":
    query = input("Ask a question about your document: ")
    context = retrieve(query)
    print("\\n--- Retrieved Context ---")
    for c in context:
        print(f"  • {c[:100]}...")
    print("\\n--- Generating Answer ---")
    answer = generate(query, context, backend="ollama")  # change to "openai" if preferred
    print(f"\\nAnswer: {answer}")
