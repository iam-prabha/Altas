RAG from Scratch

Build your first working Retrieval-Augmented Generation pipeline — with no frameworks, just TypeScript.

What you’ll build

By the end of this lesson you’ll have a rag.ts that:

  1. Loads a document and chunks it
  2. Turns each chunk into an embedding (a vector)
  3. Searches those vectors to find relevant chunks for your question
  4. Generates an answer using an LLM with the retrieved context

What is RAG?

Retrieval-Augmented Generation is a pattern that gives an LLM access to external knowledge at inference time. Instead of answering from its training data alone, the model first retrieves relevant documents, then generates an answer grounded in those documents.

Think of it as giving the LLM an open-book exam: it can look up facts before answering. Without RAG, it’s answering from memory alone — which is why it hallucinates or gives outdated information.

The three steps every RAG system follows:

Step Name What happens
1 Index Split documents into chunks, embed each chunk into a vector, store in a searchable index
2 Retrieve Embed the user’s query, find the most similar chunks via vector search
3 Generate Feed the query + retrieved chunks to an LLM, get a grounded answer

This is called “Naive RAG” — just the three steps in sequence. More advanced variants add pre/post-processing, but this is the core every system builds on.

Interview prep: You’ll often be asked to “walk through how a RAG system works.” The Index → Retrieve → Generate framework is your mental model. Know it cold.

Setup

Create a file called sample.txt with any text you want to query against. Here’s a short one to start:

Retrieval-Augmented Generation (RAG) is a technique that combines
information retrieval with text generation. It works by first retrieving
relevant documents from a knowledge base, then feeding those documents
as context to a language model to generate a grounded response.

One of the key advantages of RAG is that it reduces hallucinations by
grounding the model's output in retrieved evidence. It also enables
the system to access up-to-date information without retraining.

RAG systems typically use embeddings to represent documents and
queries as vectors, then perform similarity search to find the most
relevant documents for a given query. The retrieved documents are
then concatenated with the query and passed to the LLM for generation.

Step 1: Chunking

Step 1

Documents are too long to fit in an LLM’s context window. We split them into smaller pieces (chunks) that can be retrieved and fed to the model.

A simple chunking strategy: split by character count with overlap so no meaning is lost at boundaries.

import fs from "fs";

function chunkText(text: string, chunkSize = 500, overlap = 50): string[] {
  const chunks: string[] = [];
  let start = 0;
  while (start < text.length) {
    const end = Math.min(start + chunkSize, text.length);
    chunks.push(text.slice(start, end));
    start += chunkSize - overlap;
  }
  return chunks;
}

const text = fs.readFileSync("sample.txt", "utf-8");
const chunks = chunkText(text);
console.log(`Split into ${chunks.length} chunks`);
chunks.forEach((c, i) => {
  console.log(`--- Chunk ${i} (${c.length} chars) ---`);
  console.log(c.slice(0, 100) + "...");
});

Step 2: Embeddings

Step 2

Computers don’t understand text — they understand numbers. An embedding is a list of numbers (a vector) that captures the meaning of a piece of text. Similar texts have similar vectors.

import { pipeline } from "@xenova/transformers";

// Load a small, fast embedding model
const model = await pipeline("feature-extraction", "Xenova/all-MiniLM-L6-v2");

// Turn chunks into vectors
const result = await model(chunks, { pooling: "mean", normalize: true });
const chunkEmbeddings: number[][] = result.tolist();
console.log(`Embedding shape: [${chunkEmbeddings.length}, ${chunkEmbeddings[0].length}]`);
// [num_chunks, 384] — each chunk becomes a 384-dimensional vector

all-MiniLM-L6-v2 is a popular embedding model. It’s small (80 MB), fast, and gives decent quality. You can swap it for stronger models (e.g., BAAI/bge-large-en-v1.5) later.

Step 3: Vector Search with FAISS

Step 3

Now we need to find the chunks most relevant to a user’s question. We embed the question, then find the chunks whose vectors are closest to it. FAISS (Facebook AI Similarity Search) does this efficiently.

function cosineSimilarity(a: number[], b: number[]): number {
  let dotProduct = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}

async function retrieve(query: string, k = 2): Promise<string[]> {
  // Embed the query
  const queryResult = await model([query], { pooling: "mean", normalize: true });
  const queryVec: number[] = queryResult.tolist()[0];

  // Find k most similar chunks by cosine similarity
  const similarities = chunkEmbeddings.map(emb => cosineSimilarity(queryVec, emb));
  const topK = similarities
    .map((sim, i) => ({ sim, i }))
    .sort((a, b) => b.sim - a.sim)
    .slice(0, k)
    .map(x => x.i);

  return topK.map(i => chunks[i]);
}

// Try it!
const results = await retrieve("How does RAG reduce hallucinations?");
console.log("Retrieved chunks:");
for (const r of results) {
  console.log(`  • ${r.slice(0, 80)}...`);
}

L2 (Euclidean distance) is the simplest similarity metric. In practice, cosine similarity is more common for text embeddings — we’ll cover that in a later lesson.

Step 4: Generation

Step 4

Finally, we feed the retrieved context + the original question to an LLM. The LLM uses the context to generate a grounded answer.

Option A: Ollama (local, free)

async function generateOllama(
  query: string,
  context: string[],
  model = "llama3.2"
): Promise<string> {
  const prompt = `Answer the question using ONLY the provided context.

Context:
${context.join("\n")}

Question: ${query}

Answer:`;
  const response = await fetch("http://localhost:11434/api/generate", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ model, prompt, stream: false }),
  });
  const data = await response.json();
  return data.response;
}

// Usage
const context = await retrieve(query);
const answer = await generateOllama(query, context);
console.log("Answer:", answer);

Option B: OpenAI API

import OpenAI from "openai";

const client = new OpenAI();  // requires OPENAI_API_KEY env var

async function generateOpenAI(
  query: string,
  context: string[],
  model = "gpt-4o-mini"
): Promise<string> {
  const prompt = `Answer the question using ONLY the provided context.\n\nContext:\n${context.join("\n")}\n\nQuestion: ${query}\n\nAnswer:`;
  const response = await client.chat.completions.create({
    model,
    messages: [{ role: "user", content: prompt }],
  });
  return response.choices[0]!.message.content ?? "";
}

// Usage
const answer = await generateOpenAI(query, context);
console.log("Answer:", answer);

The prompt instructs the LLM to “answer using ONLY the provided context.” This is critical — without it, the model may fall back on its training data and hallucinate.

Putting It All Together

Here’s the full rag.ts:

import fs from "fs";
import readline from "readline";
import { pipeline } from "@xenova/transformers";

const rl = readline.createInterface({ input: process.stdin, output: process.stdout });
const prompt = (q: string) => new Promise<string>(resolve => rl.question(q, resolve));

// --- Step 1: Load & Chunk ---
function loadAndChunk(path: string, chunkSize = 500, overlap = 50): string[] {
  const text = fs.readFileSync(path, "utf-8");
  const chunks: string[] = [];
  let start = 0;
  while (start < text.length) {
    const end = Math.min(start + chunkSize, text.length);
    chunks.push(text.slice(start, end));
    start += chunkSize - overlap;
  }
  return chunks;
}

// --- Step 2: Embed ---
console.log("Loading embedding model...");
const model = await pipeline("feature-extraction", "Xenova/all-MiniLM-L6-v2");
const chunks = loadAndChunk("sample.txt");
const result = await model(chunks, { pooling: "mean", normalize: true });
const chunkEmbeddings: number[][] = result.tolist();
console.log(`Indexing ${chunks.length} chunks...`);

// --- Step 3: Search ---
function cosineSimilarity(a: number[], b: number[]): number {
  let dotProduct = 0, normA = 0, normB = 0;
  for (let i = 0; i < a.length; i++) {
    dotProduct += a[i] * b[i];
    normA += a[i] * a[i];
    normB += b[i] * b[i];
  }
  return dotProduct / (Math.sqrt(normA) * Math.sqrt(normB));
}

async function retrieve(query: string, k = 2): Promise<string[]> {
  const queryResult = await model([query], { pooling: "mean", normalize: true });
  const queryVec: number[] = queryResult.tolist()[0];
  const similarities = chunkEmbeddings.map(emb => cosineSimilarity(queryVec, emb));
  const topK = similarities
    .map((sim, i) => ({ sim, i }))
    .sort((a, b) => b.sim - a.sim)
    .slice(0, k)
    .map(x => x.i);
  return topK.map(i => chunks[i]);
}

// --- Step 4: Generate ---
async function generate(
  query: string,
  context: string[],
  backend: "ollama" | "openai" = "ollama"
): Promise<string> {
  const promptText = `Answer using ONLY the context.\n\nContext:\n${context.join("\n")}\n\nQuestion: ${query}\n\nAnswer:`;

  if (backend === "ollama") {
    const resp = await fetch("http://localhost:11434/api/generate", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ model: "llama3.2", prompt: promptText, stream: false }),
    });
    const data = await resp.json();
    return data.response;
  } else {
    const { default: OpenAI } = await import("openai");
    const client = new OpenAI();
    const resp = await client.chat.completions.create({
      model: "gpt-4o-mini",
      messages: [{ role: "user", content: promptText }],
    });
    return resp.choices[0]!.message.content ?? "";
  }
}

// --- Run ---
const query = await prompt("Ask a question about your document: ");
const context = await retrieve(query);
console.log("\n--- Retrieved Context ---");
for (const c of context) {
  console.log(`  • ${c.slice(0, 100)}...`);
}
console.log("\n--- Generating Answer ---");
const answer = await generate(query, context, "ollama");  // change to "openai" if preferred
console.log(`\nAnswer: ${answer}`);
rl.close();
