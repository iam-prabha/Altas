Lesson 5: Type Manipulation

Lesson 5 of ? · ~25 minutes

So far you've written types. Now you'll learn to derive them — create new types from existing ones. This is where TypeScript becomes a genuine tool for making illegal states unrepresentable.

typeof — Grab a Value's Type

At the type level, typeof extracts the type from a runtime value.

const config = { url: "https://api.example.com", timeout: 5000 };

// Derive the type from the value
type Config = typeof config;
// { url: string; timeout: number; }

This is invaluable for inferring types from constants or libraries where types aren't exported.

keyof — Get the Keys of a Type

keyof T returns a union of the keys (property names) of type T.

interface User {
  id: number;
  name: string;
  email: string;
}

type UserKeys = keyof User;
// "id" | "name" | "email"

Useful for writing functions that operate on object properties safely:

function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user: User = { id: 1, name: "Alice", email: "a@b.com" };
getProperty(user, "name");  // type: string
getProperty(user, "id");    // type: number
// getProperty(user, "age"); // Error! "age" not in keyof User

Indexed Access Types — T[K]

You can access a property's type using bracket notation, just like at runtime:

type UserNameType = User["name"];  // string
type UserIdType = User["id"];      // number
type NameOrEmail = User["name" | "email"];  // string
Indexed access + keyof = building block for all advanced type manipulation. Every utility type is built from these primitives.

Mapped Types — Transform Every Property

A mapped type iterates over keys and transforms each property. The syntax looks like a type-level for...in loop: 

type Readonly<T> = {
  readonly [K in keyof T]: T[K];
};

type Optional<T> = {
  [K in keyof T]?: T[K];
};

// Usage
type ReadonlyUser = Readonly<User>;
// { readonly id: number; readonly name: string; readonly email: string; }

You can also transform the value type:

type Nullable<T> = {
  [K in keyof T]: T[K] | null;
};

Conditional Types — Type-Level if

Conditional types select between two types based on a condition:

type IsString<T> = T extends string ? "yes" : "no";

type A = IsString<"hello">;  // "yes"
type B = IsString<42>;       // "no"

A practical pattern — extracting the resolved type from a Promise:

type Unwrap<T> = T extends Promise<infer U> ? U : T;

type A = Unwrap<Promise<string>>;  // string
type B = Unwrap<number>;            // number (passes through)

The infer keyword introduces a new type variable that captures the inner type of Promise<U>.

Putting It Together: A Deep Partial

Here's a recursive mapped type that makes all nested properties optional:

type DeepPartial<T> = T extends object
  ? { [K in keyof T]?: DeepPartial<T[K]> }
  : T;

interface Config {
  server: { host: string; port: number };
  db: { url: string; poolSize: number };
}

type PartialConfig = DeepPartial<Config>;
// { server?: { host?: string; port?: number }; db?: { url?: string; poolSize?: number } }

Exercise: Type-Safe Event Emitter

Open src/0005-event-emitter.ts and implement a type-safe event emitter using mapped types, keyof, and conditional types.

Requirements:

  1. Define an EventMap interface that maps event names to their payload types: 
    interface MyEvents {
  userLogin: { userId: string };

  error: { message: string; code: number };

  2. Define type Listener<T> = (payload: T) => void
  3. Implement a generic EventEmitter<T> class with methods:
    • on<K extends keyof T>(event: K, listener: Listener<T[K]>): void
    • emit<K extends keyof T>(event: K, payload: T[K]): void
  4. Test by creating an emitter of MyEvents, subscribing, and emitting
