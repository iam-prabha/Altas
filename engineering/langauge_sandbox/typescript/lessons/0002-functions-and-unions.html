Function Signatures — Beyond the Basics

A function signature declares the types of its parameters and return value. TypeScript checks every call site against that signature.

Required Parameters

function add(a: number, b: number): number {
  return a + b;
}

Both a and b are required. Calling add(1) is a compile error.

Optional Parameters

Use ? to mark a parameter as optional. Its type becomes T | undefined.

function greet(name: string, greeting?: string): string {
  return `${greeting ?? "Hello"}, ${name}!`;
}

greet("Alice");               // "Hello, Alice!"
greet("Alice", "Hi");         // "Hi, Alice!"

Default Parameters

Defaults remove the | undefined — the param is always the given type.

function createUrl(path: string, base: string = "https://api.example.com"): string {
  return `${base}${path}`;
}

Rest Parameters

Collect remaining arguments into an array using ....

function sumAll(...nums: number[]): number {
  return nums.reduce((a, b) => a + b, 0);
}

sumAll(1, 2, 3);  // 6

Union Types

A union type says "this value can be either type A or type B". Written with the pipe symbol: A | B. 

function formatId(id: string | number): string {
  return `ID: ${id}`;
}

formatId(42);        // "ID: 42"
formatId("abc");     // "ID: abc"
// formatId(true);   // Error! boolean not allowed

Unions work with complex types too:

type Success = { status: "success"; data: string };
type Error = { status: "error"; message: string };
type Result = Success | Error;
Discriminated unions (like the status field above) are the bread and butter of type-safe state machines. Every variant has a literal type field you can switch on. More on this below.

Type Narrowing

When you have a union type, TypeScript won't let you access properties that don't exist on all members of the union. You need to narrow the type first — tell TS which specific member you're dealing with.

typeof narrowing

function double(value: string | number): string | number {
  if (typeof value === "string") {
    return value + value;     // narrowed to string
  }
  return value * 2;           // narrowed to number
}

double("hi");  // "hihi"
double(3);     // 6

Discriminated union with switch

type Circle = { kind: "circle"; radius: number };
type Square = { kind: "square"; side: number };
type Shape = Circle | Square;

function area(shape: Shape): number {
  switch (shape.kind) {
    case "circle":
      return Math.PI * shape.radius ** 2;
    case "square":
      return shape.side ** 2;
  }
}

Inside each case, TypeScript narrows shape to the specific variant. If you add a new variant and forget to handle it, the noImplicitReturns (or never exhaustiveness check) will catch it.

Exhaustiveness checking — add a default case that assigns to never: 
default:
  const _exhaustive: never = shape;
  return _exhaustive;
If a new variant is added and not handled, this line will error.

Exercise: Order Status Formatter

Open src/0002-functions-and-unions.ts and implement a function that formats different order statuses into human-readable strings.

Requirements:

  1. Define a discriminated union OrderStatus with at least three variants:
    • { status: "pending"; estimatedDays: number }
    • { status: "shipped"; trackingId: string }
    • { status: "delivered"; deliveredAt: Date }
  2. Write a function formatOrderStatus(status: OrderStatus): string
  3. Use a switch on status.status to narrow and return a formatted string for each variant
  4. Add a default case with a never exhaustiveness check
  5. Call it with each variant and log the results

  • Write a function parseInput(input: string | number): string that returns "Number: X" or "String: X" using typeof narrowing.
