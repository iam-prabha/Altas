Lesson 1: Types & Type Inference

Lesson 1 of ? · ~15 minutes

Before you can build an application in TypeScript you need the one thing that makes it different from JavaScript: the type system. This lesson gets you writing typed code immediately.

What is TypeScript?

TypeScript is JavaScript with a static type checker on top. You write .ts files, the compiler checks them, and produces plain .js that runs anywhere JavaScript runs.

For app builders: a type checker catches entire categories of bugs before you run the code — wrong argument types, missing properties, null references. That's less time debugging, more time shipping.

Basic Types

Every value in TypeScript has a type. Here are the primitives you will use in every application:

Type Example values Notes
string "hello", 'world' Text data
number 42, 3.14 All numbers, no int/float distinction
boolean true, false Logical values
null null Absence of value (intentional)
undefined undefined Value not assigned yet

Type Inference vs Annotations

TypeScript can infer types automatically from the value you assign: 

let name = "Alice";        // TypeScript infers: string
let age = 30;              // infers: number
let isActive = true;       // infers: boolean

You can also annotate types explicitly — useful for function parameters and return values: 

let name: string = "Alice";
let age: number = 30;
Prefer inference for variables, annotations for function boundaries. Let TS infer variables (it's almost always right). Always annotate function parameters and return types — that's where application bugs sneak in.

Here is a typed function:

function greet(name: string): string {
  return `Hello, ${name}!`;
}

The : string after name says "this parameter must be a string". The : string after the parameter list says "this function returns a string". If you pass a number or forget the return, TypeScript catches it before you run the code.

The any Escape Hatch

TypeScript has an escape hatch: any. It disables type checking for that value. Avoid it. Every use of any is a place where a runtime bug can hide. 

let loose: any = "hello";
loose = 42;            // allowed — any lets anything through
loose.toUpperCase();   // crashes at runtime (42 has no toUpperCase)
If you find yourself reaching for any, pause. Usually a union type (string | number) or a generic is the right answer.

Your Turn: Write Your First Typed Function

Exercise: Cart Total Calculator

Open src/0001-cart-total.ts and implement a function that calculates the total price of an array of items in a shopping cart.

Requirements:

  1. Define a type alias CartItem with name: string and price: number
  2. Write a function calculateTotal(items: CartItem[]): number
  3. Use .reduce() to sum all prices
  4. Call it with the sample cart below and log the result

Starter code — copy this into your file:

type CartItem = {
  name: string;
  price: number;
};

function calculateTotal(items: CartItem[]): number {
  // Your code here
}

const cart: CartItem[] = [
  { name: "Laptop", price: 1200 },
  { name: "Mouse", price: 25 },
  { name: "Keyboard", price: 80 },
];

console.log(calculateTotal(cart));

Run it: npx tsx src/0001-cart-total.ts (from the src/ directory)
