Lesson 4: Async TypeScript

Lesson 4 of ? · ~20 minutes

Every real application talks to the outside world — APIs, databases, files. That means async code. TypeScript tracks the types of async values just as strictly as synchronous ones.

Promise<T> — The Async Generic

Promise is a generic type built into the language. A Promise<string> will eventually resolve to a string. A Promise<User> to a User. 

// A function that returns a promise of a string
function fetchGreeting(): Promise<string> {
  return Promise.resolve("Hello!");
}

// An async function — the return type is automatically Promise<T>
async function fetchGreetingAsync(): Promise<string> {
  return "Hello!";
}
async functions always return a Promise. The compiler wraps whatever you return in Promise<T> automatically. async function x(): number would be an error — the return type must be Promise<number>.

Awaiting — Unwrapping the Promise

await unwraps the Promise: Promise<T> becomes T inside the async function. 

async function greet(): Promise<string> {
  const greeting: Promise<string> = Promise.resolve("Hello");
  const value: string = await greeting;  // unwrapped
  return value;
}

Typed fetch

The browser's fetch API returns Promise<Response>. You need to call .json() to get the body — but .json() returns Promise<any>. Don't stop there — type it. 

interface User {
  id: number;
  name: string;
  email: string;
}

async function getUser(id: number): Promise<User> {
  const res = await fetch(`https://api.example.com/users/${id}`);
  const user: User = await res.json();  // typed!
  return user;
}
await res.json() returns any by default. Assign it to a typed variable (or use a Response generic wrapper) to get type safety from the API response.

Error Handling with Async

Using try/catch with your Result pattern from Lesson 3 is a clean way to handle failures: 

type Result<T, E = Error> =
  | { ok: true; value: T }
  | { ok: false; error: E };

async function fetchUser(id: number): Promise<Result<User>> {
  try {
    const res = await fetch(`https://api.example.com/users/${id}`);
    if (!res.ok) return { ok: false, error: new Error(`HTTP ${res.status}`) };
    const user: User = await res.json();
    return { ok: true, value: user };
  } catch (e) {
    return { ok: false, error: e instanceof Error ? e : new Error(String(e)) };
  }
}

Parallel Async: Promise.all & Promise.allSettled

These utilities are also typed via generics. Promise.all returns a tuple of resolved types: 

async function getUsers(): Promise<[User, User]> {
  const [a, b] = await Promise.all([
    fetchUser(1),
    fetchUser(2),
  ]);
  return [a, b];
}

Promise.allSettled returns an array of { status: "fulfilled"; value: T } | { status: "rejected"; reason: any } — a built-in discriminated union.
