# Mission: TypeScript for Application Development

## Why
Build any application with confidence using TypeScript. The goal is to reach a point where TypeScript is the default tool for any project — from CLI tools to web APIs to full-stack apps — with the type system working as a reliable partner, not a fight.

## Success looks like
- Write idiomatic TypeScript without reaching for JavaScript crutches (`any` escaping, unchecked casts, no-implicit-any files)
- Read and contribute to any TypeScript codebase with full understanding of generics, discriminated unions, and type narrowing
- Design type-safe APIs, data models, and state machines that make illegal states unrepresentable
- Build and ship a real application (TBD: your first target app) solo, from scratch

## Constraints
- Steeper learning curve preferred — JS concepts embedded in TS practice, not separated
- Available runtimes: Node 25, Bun, Deno — all installed
- Self-paced, project-driven: every lesson produces runnable code
- No strict deadline — mastery over speed

## Out of scope
- Frontend frameworks (React/Vue/Svelte) for now — pure TypeScript on the backend first
- Build tooling deep dives (Webpack, Vite internals) — keep it simple with `tsx`
- Advanced type gymnastics for their own sake — pragmatic application TypeScript first
